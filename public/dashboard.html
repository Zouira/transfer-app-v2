Processus Transfert

1. Saisie du transfert pending
L'opérateur saisit le transfert : client, date, heure, départ, destination. Statut initial : En attente.
2. Assignation chauffeur assigned
L'opérateur ou le directeur assigne un chauffeur disponible au transfert.
⏰
T − 1h30 : Rappel automatique envoyé au chauffeur — doit confirmer GO
T − 1h00 : Si pas de GO → Alerte manager 🔴
3. Sur place / Confirmé confirmed
Chauffeur confirme sa présence au point de départ (appel téléphonique → opérateur met à jour, ou bouton dans l'app).
4. En cours — GO in_progress
Client à bord — bouton GO appuyé par l'opérateur ou confirmé par le chauffeur. Le trajet s'affiche sur la carte en pointillés verts.
⚠️
GO + 2h : Si toujours En cours → Alerte opérateur — vérifier le statut du transfert
5. Terminé — FIN completed
Le chauffeur confirme la fin de mission (appel → opérateur) ou l'opérateur appuie FIN dans l'app. Mission archivée.
Règles opérationnelles
🔴 Alerte T-1h : pas de GO → manager averti immédiatement
🟠 Alerte GO+2h : vérifier si le transfert est réellement terminé
❌ Annulation : statut cancelled, aucune notification envoyée
📊 Export CSV : Admin & Directeur uniquement
